# Frontend Visual Design

## Decide before coding
Write a one-sentence visual direction covering:
- Product character.
- Information density.
- Typography personality.
- Color strategy.
- Motion restraint.

Reuse the existing product language unless the task explicitly requests a redesign.

## Composition
- Establish one clear primary action or focal region.
- Use hierarchy through scale, spacing, weight, and contrast.
- Choose a consistent density; do not mix oversized marketing spacing with dense tool UI.
- Use whitespace to group meaning, not merely to look minimal.
- Make repeated surfaces follow a visible system.

## Avoid generic output
Do not default to:
- Purple-blue gradients.
- Glass cards everywhere.
- Excessive rounded rectangles.
- Random decorative blobs.
- Generic dashboard cards without information hierarchy.
- Animation that delays work.

## Refinement
Inspect typography, alignment, icon consistency, empty states, hover/focus states, truncation, and visual balance at multiple window sizes.

## Routing
Pair with `css-design-system` to implement and `accessible-responsive-ui` to verify usability.
